# Context

本项目是一个面向慢病患者的用药管理 Web 应用，核心功能架构：

- 多角色协同与交互
  双端设计：
  患者端：大字号、极简交互。以垂直时间轴展示今日任务；服药提醒采用\*\*滑动确认（Swipe-to-Dismiss/Confirm）\*\*弹窗，防止误触。
  家属端：监控视角。同步服药进度、库存状态，并在患者漏服（超过1小时）或反馈严重副作用时接收分级红色预警。
- 智能化录入与库存
  理解式 OCR：调用大语言模型（Vision API，如 DeepSeek/Kimi）识别药盒或处方照片，自动提取药名、剂量、频次，并将“一日x次”自动转化为具体时间轴点位。
  动态库存：基于打卡记录动态扣减库存，实现“库存不足+复诊提醒”的联动。支持新旧药冲突检测。
- 临床档案 (SOAP 框架)
  S (Subjective)：患者主观症状及副作用反馈（分级打分+表情勾选）。
  O (Objective)：血压、血糖等体征指标。
  A/P (Assessment/Plan)：依从性统计与自动生成的医生复诊参考报告。

\#拆解原则（必须严格遵守）

1. 接口契约先行：第一个任务必须是"架构初始化"，定义所有模块间的接口、类型、数据库 Schema，其他任务不得修改这些契约文件
2. INVEST 原则：每个任务必须独立(Independent)、有价值(Valuable)、可测试(Testable)
3. 依赖显式化：明确标注每个任务依赖哪些任务完成后才能开始
4. 单一职责：每个 worktree 只负责一个明确的功能模块
5. 完整可交付：每个任务完成后，对应功能必须可独立测试
